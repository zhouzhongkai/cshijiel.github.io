2014年11月10日

  1. HTTP协议:
    定义浏览器与web服务器如何通信和通信的数据格式
  2. HTTP协议通信过程:
    1).浏览器与服务器建立连接
    2).浏览器向web服务器发送请求数据包
    3).web服务器接受请求包，返回响应数据包
    4).浏览器接受响应包，关闭连接
  3. 请求包Request:
    1).请求行---请求方式 资源路径 协议版本
    2).消息头---key:value
    3).实体内容---请求为POST时存在表单数据
  4. 响应包Response:
    1).状态行---协议版本 状态码 描述内容
    2).消息头---key:value
    3).实体内容---逐行解析
  5. REST的设计准则:
    1).web中的所有事物抽象成资源
    2).每个资源对应唯一的标识，目录结构式的URI
    3).任何操作都不会改变资源标识
    4).任何操作都是无状态的
    5).传输使用XML、JSON或两者兼具
  6. 在REST中，使用不同的HTTP请求方法对资源进行CRUD操作，这样一来，对一个资源的不同操作就可以不用各自的URI了，只需要改变它的请求方式就可以了。
    C-->使用POST，在服务器上创建资源
    R-->使用GET，检索服务器上的资源
    U-->使用PUT，更新服务器上的资源
    D-->使用DELETE，删除服务器上的某个资源
  7. 状态码:1xx:通知 2xx:成功 3xx:重定向 4xx:客户端错误 5xx:服务器端错误
  8. HTTP协议的几种请求方法:
    GET、HEAD、POST、PUT、DELETE、OPTIONS、TRACE
    GET:发送请求获取资源数据
    HEAD:仅获取HTTP头信息
    POST:向服务器提交数据
    PUT:向服务器发送数据并且指定了存放位置
    DELETE:删除某个资源
    OPTIONS:获取URL支持的方法
    TRACE:回显服务器收到的请求
  9. JSON:轻量级的数据交换格式，与XML相比，速度更快，文档更小
  10. JSON进行数据交换的两种结构:
    表示一个对象
    var obj = {'id':1,'name':'xxx','age':21};
    var name = obj.name;
    表示一个数组
    var jsonArr = [{'id':1,'name':'zhangsan'},{'id':2,'name':'lisi'}];
    var name = jsonArr[1].name;
  11. Java转换成JSON:
    对象的转换
    Employee emp = new Emploee(1,"zhangsan","男");
    JSONObject obj = JSONObject.fromObject(emp);
    String jsonStr = obj.toString();
    数组的转换
    List emps = new ArrayList();
    for(int i=0;i<3;i++){
      Employee e = new Employee();
      e.setId(i+1);
      e.setName("name"+i);
      e.setGender("男");
      emps.add(e);
    }
    JSONArray jsonArr = JSONArray.fromObject(emps);
    String jsonStr = jsonArr.toString();

  12. Q:在服务器与服务器之间进行一些交易充值等操作时，操作成功时会显示请求地址，当然地址 也会被一些人通过非法的途径抓取到，此时如果有人模拟一个服务器向充值服务器发出恶意请 求，就会造成很大的财产损失，所以需要在进行充值的两个服务器之间添加某种验证。

    A:我们可以在进行交互的两个服务器上各自都约定好保存一致的Key，然后对于请求地址进行 加密，加密方式有MD5哈希加密、AES对称加密、RSA非对称加密
