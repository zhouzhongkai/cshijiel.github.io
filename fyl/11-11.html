2014年11月11日

  1. Token了解:
    Token是在struts2框架中使用的令牌，通常我们会在jsp页面中增加一块隐藏域用来放置Token标签，标签会在页面产生一个token id，在struts2配置文件中为token拦截器加入需要拦截的方法。jsp页面发出请求时会生成令牌保存在session(不同的页面可以共享)中。如果有action方法被拦截到，会判断session中的token id和发过来的token id是否相等。如果不相等，会跳转到指定页面，此时token id不会变；如果相等了，表示重复提交了，session中的token id会被清空等待下一次生成令牌。
  2. 设计一个用户表:
    用户登录数据表 
    /*==============================================================*/
    /* Table: user_login_tb                                         */
    /*==============================================================*/
    create table user_login_tb (
       user_id              int                  identity,
       user_account         varchar(20)          not null,
       user_password        varchar(20)          not null,
       user_key             varchar(10)          not null,
       constraint PK_USER_LOGIN_TB primary key (user_id)
    )

    用户信息数据表 
    /*==============================================================*/
    /* Table: user_info_tb                                          */
    /*==============================================================*/
    create table user_info_tb (
       info_id              int                  identity,
       user_id              int                  not null,
       info_nickname        varchar(10)          not null,
       info_gender          bit                  not null,
       info_phone           varchar(20)          not null,
       info_email           varchar(50)          not null,
       info_question        varchar(20)          not null,
       info_answer          varchar(20)          not null,
       constraint PK_USER_INFO_TB primary key (info_id)
    )

    登录日志表 
    /*==============================================================*/
    /* Table: user_login_log                                        */
    /*==============================================================*/
    create table user_login_log (
       log_id               int                  identity,
       user_id              int                  not null,
       log_ip_addr          varchar(20)          not null,
       log_login_time       smalldatetime        not null,
       constraint PK_USER_LOGIN_LOG primary key (log_id)
    )

  3. Maven了解:
    Maven是对项目架构的管理，更关键的是对包的管理。
    Maven仓库->search.maven.org
    由于掉用官方仓库的包会比较慢，企业一般都会使用自己的私有仓库。
  4. 项目架构:
    UI层->处理页面逻辑、控制器
    service层->处理业务逻辑，调用网络组件静态类
    BLL层->处理业务逻辑，调用非网络组件静态类
    Dao层->负责访问数据库
