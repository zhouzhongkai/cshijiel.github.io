Git学习：

  1. 对版本控制的简单了解。

    到远程库。

    绑定账户
    git config –global user.name "fanyongliang"
    绑定邮件
    git config –global user.email fanyongliang@gyyx.cn

    ssh-keygen –t rsa –C "fanyongliang@gyyx.cn"

    克隆一个库
    git clone git@github.com:cshijiel/cshijiel.github.io.git
    修改index.html文件并将文件放到暂存区
    git add index.html
    查看改动的地方
    git status
    提交修改
    git commit –m "Commited by Fanyongliang."
    推送数据到远程库
    git push origin master

Git 学习小结
				
1--配置git：
	$ git config --global user.name "Your Name"
	$ git config --global user.email "email@example.com"
2--创建版本库
   创建目录：$ mkdir learngit		
	创建版本库；
	初始化Git仓库，$ git init 
	添加文件到Git仓库，分两步：
	第一步，$ git add + name ，注意，可反复多次使用，添加多个文件
	第二步，$ git commit，完成
	-m "添加注释"
3--查看工作区状态
	$ git status
	$ git diff 查看修改内容
4--版本回退
	HEAD指针 指向当前版本,上一个版本就是HEAD^ ，上上一个版本就是HEAD^^ ，上100个版本 HEAD~100
	回到指定版本：
	$ git reset --hard commit_id
	$ git log 查看提交历史
	$ git log --pretty=oneline
	$ git reflog 查看命令历史
5--工作区和暂存区	
6--管理修改
	Git跟踪修改，每次修改，如果不add到暂存区，那就不会加入到commit中
7--撤销修改	
	$ git checkout -- file	
8--删除文件
	$ git rm 	
9--远程仓库
	创建SSH Key： $ ssh-keygen -t rsa -C "youremail@example.com"	
10--添加远程库
	$ git remote add origin git@server-name:path/repo-name.git
	$ git push -u origin master 第一次推送master分支的所有内容
	此后$ git push origin master 推送最新修改
11--从远程库克隆
	$ git clone git@github.com:michaelliao/gitskills.git
	Git支持多种协议，包括https，但通过ssh支持的原生git协议速度最快
	地址例子
	git://github.com/michaelliao/gitskills.git
	https://github.com/michaelliao/gitskills.git	
12--创建与合并分支
	查看分支：git branch
	创建分支：git branch + name
	切换分支：git checkout + name
	创建+切换分支：git checkout -b + name
	合并某分支到当前分支：git merge + name
	删除分支：git branch -d + name	
13--git fetch和git pull的区别

HTTP 学习小结
				
	郭海(开发) 2014-11-10 10:01:29
	B组同学今天看一下HTTP协议部分，主要了解以下几个关键词
	json ,提交数据的几种方法, rest

1--HTTP-Hypertext transfer protocol-超文本传输协议
   HTTP是无连接：无连接的含义是限制每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。采用这种方式可以节省传输时间。
   HTTP是无状态：HTTP协议是无状态协议。无状态是指协议对于事务处理没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，则它必须重传，这样可能导致每次连接传送的数据量增大。另一方面，在服务器不需要先前信息时它的应答就较快。

  -HTTP请求方法
   1-GET     请求指定的页面信息，并返回实体主体。
   2-HEAD    类似于get请求，只不过返回的响应中没有具体的内容，用于获取报头
   3-POST    向指定资源提交数据进行处理请求（例如提交表单或者上传文件）。数据被包含在请求体中。POST请求可能会导致新的资源的建立和/或已有资源的修改。
   4-PUT     从客户端向服务器传送的数据取代指定的文档的内容。
   5-DELETE  请求服务器删除指定的页面。
   6-CONNECT HTTP/1.1协议中预留给能够将连接改为管道方式的代理服务器。
   7-OPTIONS 允许客户端查看服务器的性能。
   8-TRACE   回显服务器收到的请求，主要用于测试或诊断。
	
  -HTTP 消息结构
   HTTP是基于客户端/服务端（C/S）的架构模型。
   HTTP使用统一资源标识符（Uniform Resource Identifiers, URI）来传输数据和建立连接。
   客户端请求消息
   客户端发送一个HTTP请求到服务器的请求消息包括以下格式：请求行（request line）、请求头部（header）、空行和请求数据4个部分组成。
   服务器响应消息
   HTTP响应由三个部分组成，分别是：状态行、消息报头、响应正文。
	
  -HTTP状态码
   200 - 请求成功
   301 - 资源（网页等）被永久转移到其它URL
   404 - 请求的资源（网页等）不存在
   500 - 内部服务器错误
	
2--JSON：JavaScript 对象表示法（JavaScript Object Notation）
	JSON 是存储和交换文本信息的语法
	语法规则：
		数据在名称/值对中
		数据由逗号分隔
		花括号保存对象
		方括号保存数组
	
	把 JSON 文本转换为 JavaScript 对象
	JSON 实例 - 来自于符串的 对象
	创建包含 JSON 语法的 JavaScript 字符串：
	var txt = '{ "employee":[' +
	'{ "firstName":"Bill" , "lastName":"Gates" },' +
	'{ "firstName":"George" , "lastName":"Bush" },' +
	'{ "firstName":"Thomas" , "lastName":"Carter" } ]}';
	上面的结构是这样：
	var txt = '{ "employee":[ {"":"","":""},{"":"","":""},{"":"","":""} ] }'

3--rest-Representational State Transfer-REST-表述性状态转移-是一种软件架构风格
   维基百科-wikipedia
   注意区别应用的状态和连接协议的状态。HTTP连接是无状态的（也就是不记录每个连接的信息），而REST传输会包含应用的所有状态信息，因此可以大幅降低对HTTP连接的重复请求资源消耗。
	
4--URI-统一资源标识符-Uniform Resource Identifier-一个用于标识某一互联网资源名称的字符串
   URL-统一资源定位符-Uniform / Universal Resource Locator-是因特网上标准的资源的地址（Address）
   URN-统一资源名称--统一资源标识（URI）的历史名字
