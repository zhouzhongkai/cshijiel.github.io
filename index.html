整个过程：

  1. 在本地创建git版本库
  2. 在github上创建空库，类似于username.github.io
  3. 在本地创建ssh-keygen，如下:
    ssh-keygen -t rsa -C "user.email"
    得到ssh-key后填入github.com,
    验证:
    ssh -T git@github.com
  4. 在本地执行
    git clone ...
    克隆整个项目
  5. 在本地编辑修改/增加 index.html文件，
  6. 提交
    git push origin master
  7. 以后更新先使用 
    git fetch origin master
    git pull origin master
    然后本地编辑修改提交后再push到远程库

  1. 了解和使用Git
    Git是分布式的版本控制工具，可以指定和若干不同的远端代码仓库进行交互，就可以在同一个项目中，
    分别和不同工作小组的人相互协作。Git 只关心文件数据的整体是否发生变化，Git 不会保存前后变化的差异数据。
    Git 更像是把变化的文件作快照后，记录在一个微型的文件系统中。
  2. 在学习中所使用和实验的指令及对其的理解：
    1 git config --global user.name "zhouzhongkai"

    判断权限问题，commit其他任何的人的版本库 没有验证 不就提交上去了吗...
    2 git config --list
    检查配置信息
    3 git init
    初始化，并且生成一个.git目录
    4 ssh-keygen -t rsa -C “zhouzhongkai@gyyx.cn”
    获取ssh key，发现的问题：版本库的SSH方式和HTTPS方式是不同的，认证机制不同。
    当建立了本机密钥之后，使用ssh方式实际上是不需要再次认证的，而https则每次需要输入密码 。
    5 git clone [url] 服务器上有的数据克隆之后本地都有
    6 git add + 想要提交的文件名，把数据暂存
    7 git commit -m "注释的一些信息" 如果失败会回滚到add之前，成功推送修改到本地库中；
    8 git push 将本地仓库修改推送到服务器上的仓库中
    9 push与pull
    git push origin branch_name;本地到服务器

    10 git branch branch_name 新建分支 checkout切换
    11 分支 指向commit对象的可变指针 轻量 廉价 开发遇到问题会很方便

Git学习：

  1. 对版本控制的简单了解。
    所谓版本控制就是对一些东西的所有变化过程的记录，便于我们查看过去的所有改动，在项
    目的开发过程中，记录下开发的每个阶段是很有必要的。
  2. 参考《Pro Git》学习版本控制软件Git的简单操作
    Git：分布式的版本控制系统，工作时是在工作目录、暂存区以及本地仓库三者间管理文件变
    化。使用时需要手动将变化后的文件add到暂存区，然后再commit到本地仓库，最后push
    到远程库。
  3. 配置:
    绑定账户
    git config –global user.name "fanyongliang"
    绑定邮件
    git config –global user.email fanyongliang@gyyx.cn
    生成key
    ssh-keygen –t rsa –C "fanyongliang@gyyx.cn"
  4. 操作:
    克隆一个库
    git clone git@github.com:cshijiel/cshijiel.github.io.git
    修改index.html文件并将文件放到暂存区
    git add index.html
    查看改动的地方
    git status
    提交修改
    git commit –m "Commited by Fanyongliang."
    推送数据到远程库
    git push origin master

整个过程：

  1. 在本地创建git版本库
  2. 在本地工作空间中修改文件后add到暂存区 然后commit到master分支上
  3. 查看版本库中的状态 
    4. git status
  4. 查看文件状态 
    5. git giff
  5. 显示最近提交的文件状态
    6. git log
  6. 通过SSH协议连接GitHub，获取SSHKey 
  7. ssh-keygen -t rsa -C"自己的邮箱地址"
    获取后根据提示在Users/user中找到SSHKey 在GitHub中获得相应远程版本库的认可
  8. 关联本地版本库与远程版本库
    9. git remote add origin 通常是SSH协议的版本库的地址
  9. 把本地版本库中的内容推送到远程版本库master分支上
    10. git push -u origin master 一般只有在第一次推送的时候才使用-u连接下master分支
  10. 从远程版本库中clone所有内容
    11. git clone "远程版本库的地址"
  11. 创建分支
    12. git checkout -b 分支名称
  12. 查看所有分支
    13. git branch
  13. 切换到某一分支
    14. git checkout 分支名称
  14. 将分支合并到现处分支


赵柱 2014年11月9日18:42:13

Git 学习小结
				
1--配置git：
	$ git config --global user.name "Your Name"
	$ git config --global user.email "email@example.com"
2--创建版本库
   创建目录：$ mkdir learngit		
	创建版本库；
	初始化Git仓库，$ git init 
	添加文件到Git仓库，分两步：
	第一步，$ git add ，注意，可反复多次使用，添加多个文件
	第二步，$ git commit，完成
	-m "添加注释"
3--查看工作区状态
	$ git status
	$ git diff 查看修改内容
4--版本回退
	HEAD指针 指向当前版本,上一个版本就是HEAD^ ，上上一个版本就是HEAD^^ ，上100个版本 HEAD~100
	回到指定版本：
	$ git reset --hard commit_id
	$ git log 查看提交历史
	$ git log --pretty=oneline
	$ git reflog 查看命令历史
5--工作区和暂存区	
6--管理修改
	Git跟踪修改，每次修改，如果不add到暂存区，那就不会加入到commit中
7--撤销修改	
	$ git checkout -- 	file	
8--删除文件
	$ git rm 	
9--远程仓库
	创建SSH Key： $ ssh-keygen -t rsa -C "youremail@example.com"	
10--添加远程库
	$ git remote add origin git@server-name:path/repo-name.git
	$ git push -u origin master 第一次推送master分支的所有内容
	此后$ git push origin master 推送最新修改
11--从远程库克隆
	$ git clone git@github.com:michaelliao/gitskills.git
	Git支持多种协议，包括https，但通过ssh支持的原生git协议速度最快
	地址例子
	git://github.com/michaelliao/gitskills.git
	https://github.com/michaelliao/gitskills.git	
12--创建与合并分支
	查看分支：git branch
	创建分支：git branch 
	切换分支：git checkout 
	创建+切换分支：git checkout -b 
	合并某分支到当前分支：git merge 
	删除分支：git branch -d 		
13--git fetch和git pull的区别
