Spring MVC学习小结：

注解：Annotation

Spring MVC基于注解的开发
1-创建web项目
2-在spring MVC的配置文件中指定注解驱动，配置扫描器
3-@controller:标识当前类是控制层的一个具体实现
4-@requestMapping:放在方法上面用来指定某个方法的路径，当它放在类上的时候相当于命名空间需要组合方法上的requestmapping来访问。
 -@Init+方法，初始化的方法

5-右键左下角servers-new-可以新建一个server-找到Apache-找路径
6-/ui/src/main/webapp/WEB-INF/spring/appServlet/servlet-context.xml
 MVC扫描器，扫描器可以扫描到所有注解的位置；
 
 扫描器可以扫描某个包下面的所有注解

7-注解知识点
	//注解多个参数@RequestMapping(value="/test1",method=RequestMethod.post)
	//method=RequestMethod.post设置只能通过post传参传值
	//注解只有一个参数@RequestMapping("/test1")
	//return "";指定返回的位置比如 return "user/index.jsp" ，返回到user下的index.jsp页面

	-在HomeController.java中，public class HomeController的上面用RequestMapping("/user")来指定访问路径
	  注解@RequestMapping("/path")用于指定解析路径
	-在/ui/src/main/webapp/WEB-INF/spring/appServlet/servlet-context.xml
	文件中指定视图文件的位置或文件类型
	
	
	表示以/WEB-INF/views/为前缀并且以.jsp为后缀的文件为视图文件
	
8-注解方式接收参数的4种方式
  -HttpServletRequest地址栏传参   ?name=zhanzhu&sex=m&age=20
  -直接在方法的参数上指定参数的名字，可以同时指定多个
	地址栏中传参，"&"分割两个参数
	在方法的参数中(Integer age)来指定参数为int类型（当提交参数不是int类型时，会报错，因此一般使用String类型来接收参数，后台处理时判断此字符串是否为纯数字来判断参数是否正确）	
  -直接在方法的参数上指定一个对象参数，用于接收多个参数，会自动将参数封装成为一个对象
  -直接在方法的参数上指定一个数组类型参数，用于接收多个值，(String[] name)
  
  -接收日期类型数据
